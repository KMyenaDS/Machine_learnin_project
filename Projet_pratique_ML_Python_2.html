Projet Pratique : Titanic Survival Prediction¶

Auteur : Kofivi YENA

Introduction¶

Aprs avoir appris l'optimisation d'une pipeline d'apprentissage automatique, nous passons la pratique avec un jeu de donnes du monde rel.

Vous allons utiliser la validation croise et une recherche sur une grille d'hyperparamtres pour optimiser une pipeline d'apprentissage automatique.

Nous utiliserons le jeu de donnes Titanic Survival Dataset pour construire un modle de classification capable de prdire si un passager a survcu au naufrage du Titanic, en fonction des caractristiques de chaque passager dans le jeu de donnes.

Nous commencerons par construire un Random Forest Classifier, puis nous modifierons notre pipeline pour utiliser un estimateur de rgression logistique la place. Nous valuerons et comparerons nos rsultats.

Objectifs¶

Ce projet consiste :

  • Utiliser scikit-learn pour construire un modle afin de rsoudre un problme de classification
  • Mettre en uvre un pipeline combinant les tapes de prtraitement avec un modle d'apprentissage automatique
  • Interprter les rsultats de notre modlisation
  • Mettre jour notre pipeline avec un modle d'apprentissage automatique diffrent
  • Comparer les performances de nos classificateurs

Dfinir des pipelines de prtraitement distincts pour chaque type de caractristique

numerical_transformer = Pipeline(steps=[
    ('imputer', SimpleImputer(strategy='median')),
    ('scaler', StandardScaler())
])

categorical_transformer = Pipeline(steps=[
    ('imputer', SimpleImputer(strategy='most_frequent')),
    ('onehot', OneHotEncoder(handle_unknown='ignore'))
])

Combiner les transformateurs en un seul transformateur de colonnes¶

Nous utiliserons l'estimateur "column transformer" de sklearn pour transformer sparment les caractristiques, puis concatner la sortie en un seul espace de caractristiques, prt tre utilis comme entre pour un estimateur de machine learning.

In [10]:
preprocessor = ColumnTransformer(
    transformers=[
        ('num', numerical_transformer, numerical_features),
        ('cat', categorical_transformer, categorical_features)
    ])

Compltons maintenant le pipeline de modlisation en combinant le prtraitement avec un classificateur Random Forest.

pipeline = Pipeline(steps=[
    ('preprocessor', preprocessor),
    ('classifier', RandomForestClassifier(random_state=42))
])

Obtenons les importances des caractéristiques de notre modèle global.¶

Tout d'abord, pour obtenir les importances des caractristiques catgorielles, nous devons remonter travers le pipeline de modlisation afin d'associer les importances des caractristiques aux variables encodes en one-hot, qui ont t transformes partir des caractristiques catgorielles d'origine.

Nous n'avons pas besoin de remonter le pipeline pour les caractristiques numriques, car nous ne les avons pas transformes en de nouvelles variables.

Rappelons que nous avons converti les caractristiques catgorielles en caractristiques encodes en one-hot l'aide du transformateur de colonnes cat .

En pratique, il est recommand de tester diffrents modles et mme de revoir lanalyse des donnes pour amliorer les performances du modle. Il pourrait tre utile de crer de nouvelles caractristiques ou dimputer les valeurs manquantes afin dexploiter un plus grand volume de donnes.
