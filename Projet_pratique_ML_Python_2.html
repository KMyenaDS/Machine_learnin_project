Projet Pratique : Titanic Survival Prediction¶

Auteur : Kofivi YENA

Introduction¶

Après avoir appris l'optimisation d'une pipeline d'apprentissage automatique, nous passons à la pratique avec un jeu de données du monde réel.

Vous allons utiliser la validation croisée et une recherche sur une grille d'hyperparamètres pour optimiser une pipeline d'apprentissage automatique.

Nous utiliserons le jeu de données Titanic Survival Dataset pour construire un modèle de classification capable de prédire si un passager a survécu au naufrage du Titanic, en fonction des caractéristiques de chaque passager dans le jeu de données.

Nous commencerons par construire un Random Forest Classifier, puis nous modifierons notre pipeline pour utiliser un estimateur de régression logistique à la place. Nous évaluerons et comparerons nos résultats.

Objectifs¶

Ce projet consiste à :

  • Utiliser scikit-learn pour construire un modèle afin de résoudre un problème de classification
  • Mettre en œuvre un pipeline combinant les étapes de prétraitement avec un modèle d'apprentissage automatique
  • Interpréter les résultats de notre modélisation
  • Mettre à jour notre pipeline avec un modèle d'apprentissage automatique différent
  • Comparer les performances de nos classificateurs

Définir des pipelines de prétraitement distincts pour chaque type de caractéristique

In [9]:
numerical_transformer = Pipeline(steps=[
    ('imputer', SimpleImputer(strategy='median')),
    ('scaler', StandardScaler())
])

categorical_transformer = Pipeline(steps=[
    ('imputer', SimpleImputer(strategy='most_frequent')),
    ('onehot', OneHotEncoder(handle_unknown='ignore'))
])

Combiner les transformateurs en un seul transformateur de colonnes¶

Nous utiliserons l'estimateur "column transformer" de sklearn pour transformer séparément les caractéristiques, puis concaténer la sortie en un seul espace de caractéristiques, prêt à être utilisé comme entrée pour un estimateur de machine learning.

In [10]:
preprocessor = ColumnTransformer(
    transformers=[
        ('num', numerical_transformer, numerical_features),
        ('cat', categorical_transformer, categorical_features)
    ])

Créer un pipeline de modélisation¶

Complétons maintenant le pipeline de modélisation en combinant le prétraitement avec un classificateur Random Forest.

In [11]:
pipeline = Pipeline(steps=[
    ('preprocessor', preprocessor),
    ('classifier', RandomForestClassifier(random_state=42))
])

Obtenons les importances des caractéristiques de notre modèle global.¶

Tout d'abord, pour obtenir les importances des caractéristiques catégorielles, nous devons remonter à travers le pipeline de modélisation afin d'associer les importances des caractéristiques aux variables encodées en one-hot, qui ont été transformées à partir des caractéristiques catégorielles d'origine.

Nous n'avons pas besoin de remonter le pipeline pour les caractéristiques numériques, car nous ne les avons pas transformées en de nouvelles variables.

Rappelons que nous avons converti les caractéristiques catégorielles en caractéristiques encodées en one-hot à l'aide du transformateur de colonnes « cat ».

Essayons un autre modèle¶

En pratique, il est recommandé de tester différents modèles et même de revoir l’analyse des données pour améliorer les performances du modèle. Il pourrait être utile de créer de nouvelles caractéristiques ou d’imputer les valeurs manquantes afin d’exploiter un plus grand volume de données.

Grâce à la classe Pipeline de Scikit-learn, cette tâche est simple et rapide. Mettons à jour le pipeline et la grille de paramètres afin d’entraîner un modèle de régression logistique et comparer ses performances avec celles du modèle précédent.

In [23]:
